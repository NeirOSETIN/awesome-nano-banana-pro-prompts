## 🚀 如何贡献

### GitHub Issue（唯一方式）

我们仅通过 GitHub Issues 接受贡献，以确保质量控制和格式一致性。

1. 点击 [**提交新提示词**](https://github.com/YouMind-OpenLab/awesome-nano-banana-pro-prompts/issues/new?template=submit-prompt.yml)
2. 填写表单中的所有必填信息
3. 提交您的 Issue
4. 团队将在 48 小时内审核
5. 如果通过（我们会添加 `approved` 标签），将自动同步到 CMS
6. 您的提示词将在 4 小时内出现在 README 中

**为什么只接受 Issues？**
- ✅ 格式统一
- ✅ 更好的质量控制
- ✅ 自动化处理
- ✅ 降低贡献门槛

## 📝 提交指南

### 必填信息

- **标题**：清晰、描述性（最多 80 字符）
- **提示词**：完整的提示词文本
- **描述**：它的作用、何时使用
- **图片**：高质量结果（最小宽度 512px）
- **作者**：原创作者的姓名和链接
- **来源**：原帖/推文链接
- **语言**：提示词的语言

### 图片要求

- ✅ 最小宽度：512px
- ✅ 推荐：1024px - 2048px
- ✅ 格式：JPEG、PNG、WebP
- ✅ 文件大小：< 5MB
- ✅ 无水印（除原作者水印外）

### 质量标准

我们接受以下提示词：

- ✅ **原创**或正确署名
- ✅ **高质量**结果
- ✅ **清晰**且可复现
- ✅ **富有创意**且启发性强
- ✅ **工作场所安全**（无敏感内容）

我们拒绝：

- ❌ 低质量或模糊的图片
- ❌ 抄袭内容
- ❌ NSFW 内容
- ❌ 有害或冒犯性材料
- ❌ 垃圾信息或推广内容

## 📜 许可协议

通过贡献，您同意：

1. **许可**：您的贡献将在 [CC BY 4.0](https://creativecommons.org/licenses/by/4.0/) 下授权
2. **署名**：将正确署名原作者
3. **权利**：您确认有权提交此内容

## 🔄 审核流程

1. **提交** → 用户创建 GitHub Issue
2. **审核** → 团队检查质量（48 小时内）
3. **批准** → 团队添加 `approved` 标签
4. **自动同步** → 工作流触发，在 CMS 中创建提示词
5. **README 更新** → 在下一次 4 小时更新时出现在 README 中
